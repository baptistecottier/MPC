Secure Comparison based on Garbled Circuits

1. PROJECT OVERVIEW

In order to fully understand this project, one should have knowledge on garbled circuits, elleptic curves and homomorphic encryption

The folder is organized as follows :

  • bin/: Files generated during compilation
  • doc/: Technical documentation of the scheme
  • lib/: Third party libraries used
  • src/: Source code of the scheme
  • doxygen.conf: Documentation configuration file
  • Makefile: Makefile


2. INSTALLATION INSTRUCTIONS

2.1 Requirements

The following softwares and librairies are required: make, gcc, openssl and gmp.

2.2 Compilation Step

  • Execute make comparison to compile a working example of the comparison. Run bin/comparison to execute the comparison and display the result.

During compilation, the following files are created inside the bin/build folder:

  • hash.o: A wrapper around openssl SHA512 implementation
  • auxiliary_functions.o: background functions used in other functions
  • cmp_steps.o: main functions used in API and their initializations and clearences
  • gate_functions.o: functions used to garble and evaluate gates
  • oblivious_transfer.o: functions used in the oblivious transfer
  • paillier.o: functions to encrypt and decrypt with Paillier keys
  • randombytes.o: functions used to generate random inputs
  • twisted_edwards_curves.o: functions used for computations on twisted Edwards curves


3. DOCUMENTATION GENERATION

3.1 Requirements

The following softwares are required: doxygen and bibtex.

3.2 Generation Step

  • Run doxygen doxygen.conf to generate the code documentation
  • Browse doc/html/index.html to read the documentation
